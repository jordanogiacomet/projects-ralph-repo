## Codebase Patterns
- Payload CMS 3.x is integrated into Next.js (not standalone) — config at `src/payload.config.ts`
- Frontend routes live in `src/app/(frontend)/`, Payload admin/API in `src/app/(payload)/`
- Use `@payload-config` path alias to import payload config
- Use `@/*` path alias for src imports
- Tailwind design tokens: accent (#0056A6), bg-dark-section (#1A1A2E), cta-green (#28A745)
- Collections go in `src/collections/`, globals in `src/globals/`, blocks in `src/blocks/`
- Use `--legacy-peer-deps` for npm install due to Payload/Next.js peer dep conflicts
- Next.js version must match Payload peer requirements (currently 15.4.x)
- Access control helpers at `src/access/index.ts` — use `isAdmin`, `isAdminOrEditor`, `isLoggedIn`, `anyone`
- Slug auto-generation hook at `src/hooks/formatSlug.ts` — use `formatSlugHook` in field `hooks.beforeValidate`
- Collection slugs use kebab-case (e.g., `solucao-categories`, `cliente-segmentos`)
