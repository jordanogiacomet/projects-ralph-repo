## Codebase Patterns
- Payload CMS 3.x is integrated into Next.js (not standalone) — config at `src/payload.config.ts`
- Frontend routes live in `src/app/(frontend)/`, Payload admin/API in `src/app/(payload)/`
- Use `@payload-config` path alias to import payload config
- Use `@/*` path alias for src imports
- Tailwind design tokens: accent (#0056A6), bg-dark-section (#1A1A2E), cta-green (#28A745)
- Collections go in `src/collections/`, globals in `src/globals/`, blocks in `src/blocks/`
- Use `--legacy-peer-deps` for npm install due to Payload/Next.js peer dep conflicts
- Next.js version must match Payload peer requirements (currently 15.4.x)
- Access control helpers at `src/access/index.ts` — use `isAdmin`, `isAdminOrEditor`, `isLoggedIn`, `anyone`
- Slug auto-generation hook at `src/hooks/formatSlug.ts` — use `formatSlugHook` in field `hooks.beforeValidate`
- Collection slugs use kebab-case (e.g., `solucao-categories`, `cliente-segmentos`)
- Block definitions live in `src/blocks/`, exported via `src/blocks/index.ts`
- BlockRenderer component at `src/components/BlockRenderer.tsx` maps blockType to React components
- Import all blocks from `@/blocks` barrel export when registering in collections
- Server/client split for interactive layout components: FooServer.tsx (async, fetches data) + Foo.tsx ('use client', handles state/events)
- Payload upload relations may be string IDs or objects — always check `typeof field === 'object'` before accessing `.url`
- When route slugs change, keep a backward-compatible route file that re-exports the canonical page (`export { default, generateMetadata } from '../canonical/page'`)
- Social icons are inline SVGs in SocialLinks.tsx — no icon library needed
- Use framer-motion AnimatePresence for enter/exit animations on conditional renders
- Framer Motion components should gate animation intensity with `useReducedMotion()` and disable drag/transition-heavy behavior for users with reduced-motion preferences
- Overlay-style UI (mobile menu/search) should trap focus with Tab/Shift+Tab and close on Escape while open
- Frontend typography is loaded via `next/font/google` in `src/app/(frontend)/layout.tsx`; Tailwind `font-sans` maps to CSS variables (`--font-inter`, `--font-open-sans`)
- Global layout components should include deterministic fallback data when Payload globals are unavailable during static builds
- Footer defaults for unidades/social/cookie/WhatsApp are centralized in `src/lib/constants.ts` and merged with Payload globals in `src/components/Footer.tsx`
- Advanced contact forms follow a server-page + client-form split and submit JSON payloads to `/api/form-submissions`; until binary upload persistence is defined, file fields should store validated metadata in `data`
- Download-gated content flows can reuse `/api/form-submissions` with `formType: 'contato'` and extra context fields in `data` (for example `origem`, `conteudoSlug`) before releasing protected files
- Posts author relations may resolve as populated `users` objects or raw IDs, so UI mappers must guard `typeof author === 'object'` before reading `name`/`email`
- Route metadata should be authored via `buildMetadata` from `src/lib/seo.ts` to keep canonical, `hreflang` (`pt-BR`), Open Graph, Twitter Card, and icon tags consistent with global fallbacks
- Public site origin resolution should prefer `NEXT_PUBLIC_SERVER_URL` and fallback to `NEXT_PUBLIC_SITE_URL` for backward compatibility across SEO/news rendering
- Structured data scripts are emitted with `JsonLd` (`src/components/JsonLd.tsx`) and schema builders in `src/lib/seo.ts` (Organization, LocalBusiness, BreadcrumbList, Article)
- Search engine discovery routes are centralized at `src/app/sitemap.ts` and `src/app/robots.ts`; include both static paths and CMS-driven slugs with safe fallbacks when Payload is unavailable
- Global site search is served by `src/app/api/search/route.ts` and consumed by `SearchOverlay`; keep the response grouped by collection type (`pages`, `posts`, `solucoes`, `conteudos`) so the UI can render typed sections without extra client-side joins
- Payload CLI `run` imports scripts and exits when module import resolves, so async seed scripts should use top-level `await` to guarantee completion
- Nested Payload array fields in Postgres should avoid reusing the same field name at consecutive levels (`children -> children`), otherwise relation names can collide during global reads
